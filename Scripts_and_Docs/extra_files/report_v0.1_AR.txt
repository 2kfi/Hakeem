## شو هو حكيم؟ — بسيط وواضح

**حكيم** = مساعد صحي ذكي يعمل **بدون إنترنت** على أجهزة رخيصة.

يدعم:
- **الصوت بالعربي** ← تحكيه، يسمعك، يرد صوتاً
- **الأسئلة الطبية** ← تشخيص، أدوية، توعية
- **تحليل الصور الطبية** ← صور الكبد، الأشعة (مع جزئية أحمد)
- **يشتغل على Raspberry Pi** ← جهاز بـ 50$ تقريباً

### نسختان:

| | Hakeem Home | Hakeem Enterprise |
|---|---|---|
| **للمن** | الفرد والأسرة | المستشفيات والعيادات |
| **الجهاز** | Raspberry Pi / Atom PC | سيرفر متوسط |
| **الاتصال** | offline كامل | offline + cloud اختياري |
| **الواجهة** | صوت + UI بسيط | داشبورد + سجلات مرضى |

## كيف يشتغل؟ — للمرشدة

*(هون بتفصّل حسب اهتمامها — إذا سألت عن الـ AI قل لها التالي)*

### الموديل: MedGemma 4B من Google
- نموذج لغوي **متخصص بالطب** من Google، مفتوح المصدر
- حجم 4 مليار معامل — صغير بيكفي يشتغل محلياً
- **عملنا عليه Fine-tuning** مرتين على 6 datasets طبية (منها عربية)
- النتيجة: موديل خاص بنا اسمه `Hakeem-MedGemma-v2` منشور على HuggingFace

### الـ Stack التقني (للي بيحب التفاصيل)
```
المستخدم [صوت] 
    → faster-whisper (STT - عربي، offline)
        → FastAPI Server
            → llama.cpp + MedGemma Q4_K_M
                → FastMCP Tools (قاعدة بيانات، بحث)
                    → piper-tts (TTS - عربي، offline)
                        → المستخدم [يسمع الجواب]
```

## النتائج — Benchmark Results

> اختبرنا 4 نماذج على **100 سؤال طبي** من معيار USMLE العالمي

| النموذج | الدقة | ملاحظة |
|---|---|---|
| Qwen3-4B (عام) | 25% | نموذج عام بدون تخصص طبي |
| Gemma3-4B (عام) | 48% | عام لكن أفضل |
| MedGemma (base) | 50% | نموذج Google الطبي الأصلي |
| MedGemma-FT v1 | 49% | ضبط أول — عادي |
| **Hakeem v2 ✅** | **53%** | **أعلى دقة + أسرع جواب (1.96 ثانية!)** |

**الأهم**: وقت الجواب انخفض من 8.3 ثانية → 1.96 ثانية بعد الضبط الثاني!

> 🔗 الموديل على HuggingFace: `huggingface.co/2kfi/Hakeem-MedGemma-4B-MedQA-v1`

## جزئية أحمد — نظام صحة الكبد

*(هون تقول: "أحمد عنده جزء متخصص في التشخيص الطبي، خليه يشرح")*

### شو بيعمل أحمد؟

**المشكلة**: الكبد عضو صامت — السرطان يظهر متأخر

**الحل**: نظام يحلل:
1. **إنزيمات الكبد** (ALT / AST) من قاعدة بيانات المرضى → يعطيها لـ MedGemma كمدخل منظّم
2. **صور الأنسجة** → يطبق فلاتر (CLAHE, Otsu, Gaussian) لتحسين الصورة → MedGemma يحلل ويكشف الكتل المشبوهة

**الربط**: أحمد يبني MCP Tool خاص يوصّل نظامه بـ Hakeem مباشرة

> 🔗 مشروع أحمد: `github.com/AIOL-HQ/liver-health-system`

## أسئلة متوقعة من المرشدة — وجاوبات مقترحة

**س: "53% دقة — هاد كافي طبياً؟"**  
ج: للنموذج بحجم 4B هاد ممتاز. الأطباء الكبار بالعالم بيستخدموا نماذج 70B+. هدفنا مش نستبدل الطبيب — بل نعطي دعم أولي في غيابه. والدقة بتتحسن مع كل جولة ضبط.

**س: "ليش offline؟ مش أسهل تحطوا على cloud؟"**  
ج: الـ cloud يحتاج إنترنت، وفي كثير من مناطق الأزمات والريف ما في. وكمان مشكلة خصوصية البيانات الطبية — البيانات تبقى عند المريض.

**س: "شو الفرق عن ChatGPT؟"**  
ج: ChatGPT يحتاج إنترنت، ما تخصص طبي، وبياناتك بتروح لسيرفرات بعيدة. حكيم: offline، متخصص، بياناتك عندك.

**س: "شو البيانات اللي درّبتوا عليها؟"**  
ج: 6 datasets طبية — منها عربية (Shifaa Arabic Medical Consultations)، وإنجليزية (USMLE, Radiology). كل شيء موثق على GitHub.

**س: "كيف تضمنوا الأمان؟"**  
ج: WireGuard VPN بين السيرفرات، API authentication، قاعدة البيانات معزولة عن الإنترنت.

**س: "شو الخطوة القادمة إذا دعمتم المشروع؟"**  
ج: نشر على Raspberry Pi + تطوير Enterprise edition للمستشفيات + دراسة ميدانية في مجتمعات ريفية.
