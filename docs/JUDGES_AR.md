## 2. الملخص

يقدم هذا المشروع **حكيم**، مساعدًا طبيًا صوتيًا ثنائي اللغة (الإنجليزية/العربية) يتيح الوصول إلى المعلومات الطبية بدون استخدام اليدين من خلال التفاعل الصوتي الطبيعي. يجمع النظام بين التعرف على الكلام المتطور (Whisper)، ونموذج لغوي طبي مضبوط بدقه للمجال الطبي (MedGemma)، ومصدر للمعلومات بدون اتصال انترنت (ويكيبيديا عبر ملفات ZIM)، وتركيب لتحويل النص الى كلام (Piper TTS).

الابتكارات الرئيسية:
- كشف وتفعيل عن طريق كلمة تنبيه مخصصة ("حكيم")
- دعم ثنائي اللغة (الإنجليزية + العربية)
- قدرة التشغيل بدون اتصال انترنت بعد الأعداد
- تحسين المجال الطبي باستخدام MedGemma

يحقق النظام أداء تنافسيًا في معايير طرح الأسئلة الطبية مع الحفاظ على سهولة الوصول للمستخدمين ذوي المعرفة التقنية المحدودة أو موارد الأجهزة المحدودة.

---

## 3. صياغة المشكلة

### 3.1 الخلفية

في العالم العربي والمناطق النامية، يظل الوصول إلى المعلومات الطبية الموثوقة تحديًا:
- محدودية المتخصصين في الرعاية الصحية للفرد
- حواجز اللغة في المعلومات الطبية
- مشاكل اتصال الإنترنت في المناطق الريفية
- ارتفاع تكاليف الاستشارات الصحية

### 3.2 المشكلة

1. **فجوة المعلومات**: لا يستطيع كثيرون الوصول بسهولة للمعلومات الصحية الموثوقة
2. **حاجز اللغة**: معظم الموارد الطبية بالإنجليزية؛ الموارد العربية محدودة
3. **إمكانية الوصول**: التفاعل الجسدي مع أنظمة الرعاية الصحية ليس دائمًا ممكنًا
4. **الفجوة الرقمية**: الواجهات المعقدة تستبعد كبار السن أو الأقل خبرة بالتكنولوجيا

### 3.3 حلنا

يعالج حكيم هذه التحديات من خلال توفير:
- تفاعل قائم على الصوت (متاح للجميع)
- دعم ثنائي اللغة (العربية + الإنجليزية)
- قدرة العمل بدون اتصال (يعمل بدون إنترنت بعد الإعداد)
- تفعيل بسيط (كلمة التنبيه "حكيم")

## 4. النهج التقني

### 4.1 بنية النظام

يتبع نظام حكيم خط معالجة من أربع مراحل:

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   كشف       │    │   تحويل      │    │   النموذج   │    │   تحويل      │
│   كلمة      │───▶│   الكلام    │───▶│   اللغوي    │───▶│   النص       │
│   التنبيه   │    │   إلى نص    │    │   الطبي     │    │   إلى صوت    │
│              │    │  (Whisper)  │    │  (MCP)       │    │  (Piper)     │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                           │                    │                    │
                           └────────────────────┴────────────────────┘
                                                 │
                                          ┌──────┴──────┐
                                          │  FastAPI   │
                                          │   Server   │
                                          │   :8003    │
                                          └────────────┘
```

### 4.2 تفاصيل المكونات

#### أ. كشف كلمة التنبيه
- **التقنية**: openWakeWord مع نموذج ONNX مخصص
- **كلمة التنبيه**: "حكيم"
- **النموذج**: مدرب مخصص على النطق العربي والإنجليزي

#### ب. تحويل الكلام إلى نص (STT)
- **التقنية**: Faster-Whisper
- **النماذج المستخدمة**: medium (افتراضي)، small (خفيف)، large-v3 (ثقيل)
- **الميزات**: كشف النشاط الصوتي (VAD)، كشف اللغة

#### ج. النموذج اللغوي (LLM) - الأساسي: MedGemma
- **النموذج الأساسي**: MedGemma (نموذج Gemma المضبوط للمجال الطبي من Google)
- **الاستدلال**: llama.cpp للنشر المحلي (مكمّم إلى 4-bit)
- **نافذة السياق**: 4096 رمز

**مقارنة المعايير:**
| النموذج | النوع | دقة MCQ | متوسط وقت MCQ | متوسط الوقت التوليدي |
| Qwen3-4B-it                          | 25%              | 1.435                | 10.65                                |
| Gemm3-4B-it                          | 48%              | 0.909                | 10.96                                |
| MedGemma-4B-it                       | 49%              | 0.909                | 7.815                                |
| MedGemma-4B-it-Fine-tuned            | 53%              | 1.378                | 1.962                                |

#### د. استرجاع المعرفة (MCP)
- **البروتوكول**: بروتوكول سياق النموذج (MCP)
- **مصدر البيانات**: ملفات ZIM لويكيبيديا و كتب طبية
- **الأدوات**: search، get_article، list_articles، get_suggestions

#### هـ. تحويل النص إلى كلام (TTS)
- **التقنية**: Piper TTS
- **اللغات**: الإنجليزية (en_GB-cori-high)، العربية (ar_JO-kareem-medium)

### 4.3 مجموعة التقنيات

| الطبقة | التقنية |
|--------|---------|
| **الإطار** | FastAPI |
| **تشغيل ML** | ONNX Runtime, llama.cpp |
| **الحاوية** | Docker, Docker Compose |
| **الصوت** | PyAudio, wave |
| **البروتوكول** | HTTP, SSE (لـ MCP) |

## 6. مقاييس الأداء

### 6.1 نتائج المعايير

تم تقييم النظام على مجموعة بيانات طرح الأسئلة الطبية:

| **نموذج المعيار**         | **دقة الاختيار من متعدد** | **متوسط وقت الاختيار من متعدد** | **متوسط وقت الاستجابة التوليدية** |
| ------------------------- | ------------------------- | ------------------------------- | --------------------------------- |
| Qwen3-4B-it               | 25%                       | 1.435                           | 10.65                             |
| Gemm3-4B-it               | 48%                       | 0.909                           | 10.96                             |
| MedGemma-4B-it            | 49%                       | 0.909                           | 7.815                             |
| MedGemma-4B-it-Fine-tuned | 53%                       | 1.378                           | 1.962                             |

**ملاحظة:** MedGemma هو نموذجنا الأساسي للدقة الطبية. تم اختبار Qwen3-4B كخط أساس للمقارنة.

### 6.2 أداء النظام

| المقياس | القيمة |
|---------|-------|
| **وقت البدء** | 15-30 ثانية |
| **زمن الاستجابة من البداية للنهاية** | 5-15 ثانية |
| **استخدام الذاكرة** | 2-4 جيجابايت (CPU)، 4-8 جيجابايت (GPU) |
| **حجم النموذج** | ~4 جيجابايت (LLM مكمّم) |

### 6.3 ملاحظات الجودة

- **نقاط القوة**: استدلال سريع، إخراج ثنائي اللغة، قدرة العمل بدون اتصال، تحسين المجال الطبي
- **النموذج الأساسي**: MedGemma يوفر دقة طبية أفضل من النماذج العامة الغرض
- **مجالات التحسين**: سعة نموذج أكبر، بيانات تدريب أكثر

## 7. الاعتبارات الأخلاقية

### 7.1 إخلاء المسؤولية الطبية

> **مهم**: تم تطوير حكيم لأغراض **تعليمية وبحثية فقط**. ليس:
> - جهاز طبي معتمد
> - بديلاً عن المشورة الطبية المهنية
> - معتمدًا للتشخيص السريري أو العلاج
>
> يجب على المستخدمين **استشارة متخصصين مؤهلين في الرعاية الصحية** دائمًا.

### 7.2 الخصوصية

- جميع المعالجة يمكن أن تعمل محليًا (بدون اعتماد سحابي)
- بيانات الصوت تبقى على جهاز المستخدم
- لا يوجد جمع للبيانات الصحية الشخصية

### 7.3 التحيز والقيود

- النموذج مدرب على بيانات طبية مركّزة على الإنجليزية
- تغطية المفردات الطبية العربية محدودة
- قطع المعرفة يعتمد على تاريخ بيانات التدريب
- قد لا يمثل جميع الممارسات الطبية الإقليمية

## 8. القيود والعمل المستقبلي

### 8.1 القيود الحالية

1. **حجم النموذج**: مكمّم إلى 4-bit؛ النموذج الكامل سيحسّن الدقة
2. **قاعدة المعرفة**: محدود على ويكيبيديا؛ الكتب الطبية ستحسّن الجودة
3. **التقييم**: معيار واحد فقط؛ يحتاج اختبارًا أوسع
4. **كلمة التنبيه**: لغة واحدة؛ لم يتم تنفيذ كلمة تنبيه عربية
5. **لا تشخيص**: لا يمكنه تقديم تشخيصات طبية رسمية

### 8.2 العمل المستقبلي

| الأولوية | التحسين | الوصف |
|----------|---------|-------|
| **عالي** | LLM أكبر | استخدام نموذج 7B أو 8B |
| **عالي** | معرفة طبية | إضافة ملفات ZIM للكتب الطبية |
| **متوسط** | كلمة تنبيه عربية | تدريب نموذج "حكيم" |
| **متوسط** | قدرة التشخيص | إضافة أدوات تحليل الأعراض |
| **منخفض** | متعدد اللغات | دعم لغات أخرى |
| **منخفض** | تطبيق جوال | تطبيق iOS/Android |

### 8.3 قابلية التوسع

يمكن نشر النظام:
- محليًا على جهاز كمبيوتر محمول
- على خادم منزلي (Raspberry Pi إلى سطح المكتب)
- في بيئة سحابية
- على أجهزة ذكاء اصطناعي متخصصة

## 9. الخلاصة

يُظهر حكيم إمكانات صوت الذكاء الاصطناعي لتديمقراطية الوصول إلى المعلومات الطبية، خاصة في المناطق ذات الموارد الصحية المحدودة أو حواجز اللغة. من خلال الجمع بين التقنيات مفتوحة المصدر (Whisper، Piper، llama.cpp) مع تحسين المجال الطبي (MedGemma)، يحقق المشروع مساعدًا صوتيًا وظيفيًا يعمل على الأجهزة الاستهلاكية.

يعالج التصميم ثنائي اللغة (الإنجليزية/العربية) فجوة حرجة في المناطق الناطقة بالعربية، بينما تضمن قدرة العمل بدون اتصال إمكانية الوصول في المناطق ذات الاتصال المحدود بالإنترنت.

### الإنجازات الرئيسية

✅ مساعد صوتي وظيفي بتركيز طبي  
✅ دعم ثنائي اللغة (الإنجليزية + العربية)  
✅ قدرة التشغيل بدون اتصال  
✅ يعمل على الأجهزة الاستهلاكية  
✅ جاهز للنشر عبر Docker  
✅ بنية مفتوحة للتوسعة
